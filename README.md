# MEGAJOM — Mega888 / 918Kiss / Pussy888 Malaysia Portal

SEO-focused affiliate site. Static Next.js 14 (App Router) → Cloudflare Pages.
All data in this repo is curated/simulated (Layer 1 only — see project brief).

## Local development

```bash
npm install
npm run dev
# open http://localhost:3000
```

## Producing a static build

```bash
npm run build     # outputs to ./out (static HTML/JS/CSS)
```

The `out/` directory is exactly what Cloudflare Pages will serve.

## RTP live data

`data/rtp-live.json` is regenerated by `scripts/update-rtp.mjs`.

- Locally: `npm run update-rtp`
- In CI: `.github/workflows/update-rtp.yml` runs every 2 hours, commits the new file,
  and the commit triggers a Cloudflare Pages rebuild so fresh numbers land in the HTML.

## Project structure

```
app/               Next.js App Router pages (homepage + /rtp)
components/        React components (server + client)
data/              games.json, rtp-live.json, daily-tips.json, winner-names.json
lib/               data + site constants
scripts/           update-rtp.mjs (cron target)
public/images/     game thumbnails (you must add these — do not hotlink)
.github/workflows/ update-rtp.yml (2-hour cron)
```

## Deployment (GitHub → Cloudflare Pages)

1. **Create a private GitHub repo**, push this directory:

   ```bash
   git init
   git add .
   git commit -m "chore: initial scaffold"
   git branch -M main
   git remote add origin git@github.com:<your-user>/ocs8-site.git
   git push -u origin main
   ```

2. **Cloudflare Pages → Create project → Connect to Git**
   - Framework preset: **Next.js (Static HTML Export)**
   - Build command: `npm run build`
   - Build output directory: `out`
   - Node version: `20`

3. **Add your custom domain** (e.g. `megajom.net`) in Cloudflare Pages → Custom domains.

4. **Update `lib/site.ts`**:
   - `domain`, `url`, `ocs8SignupUrl`, `ocs8LoginUrl`, `downloads.*`, `telegram`, `whatsapp`

## Assets to add

Game thumbnails referenced in `data/games.json` point to `/images/games/<brand>/<slug>.png`.
Drop the actual PNGs into `public/images/games/<brand>/` — a missing image just renders as
an empty tile.

Also required:
- `public/logos/ocs8.png` (OG + schema logo, 1024×1024)
- `public/favicon.ico` (already scaffolded; replace with branded icon)
- `public/og.jpg` (1200×630 share image)
